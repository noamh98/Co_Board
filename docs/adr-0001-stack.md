# ADR-0001 — הכרעת Stack ל-MVP

- **תאריך:** 2026-06-19
- **סטטוס:** מאושר
- **הקשר:** ה-PRD/HANDOFF המליצו Flutter לטווח הארוך. נדרשה הכרעה מעשית ל-MVP.

## ההחלטה
ה-MVP ייבנה כ-**PWA מבוסס React 18 + TypeScript + Vite**, offline-first
(vite-plugin-pwa / Workbox). אנדרואיד יתקבל בהמשך כעטיפה (Capacitor/TWA); iOS בפאזה 2.

## נימוק
- **יעד ה-MVP ממילא הוא "Web (PWA) תחילה"** (PRD §11.1, §12) — PWA הוא הנתיב הישיר.
- **רץ ונבדק מיד בדפדפן** — לולאת משוב מהירה; אין צורך ב-toolchain מובייל כבד.
- **offline-first טבעי** — Service Worker + IndexedDB.
- **RTL עברי** — נתמך מלא ב-DOM/CSS.
- **הסיכון הטכני המרכזי (TTS עברי אופליין) זהה בכל סטאק** — כולם נשענים על
  קולות ה-OS לבסיס; Almagu/פרימיום נשארים מקוונים. PWA לא מחמיר את הסיכון.

## חלופות שנשקלו
- **Flutter** — קוד אחד ל-Android+Web+iOS; מצוין לטווח ארוך, אך toolchain כבד
  ומשוב איטי יותר ל-MVP. נדחה ל-MVP, נשאר אופציה לאפליקציה ה-native.
- **React Native + PWA** — שני נתיבי רינדור לתחזוקה. נדחה (מורכבות).

## השלכות
- שכבת ה-domain נכתבת ב-TS טהור ובלתי-תלוית-UI → ניתנת למחזור אם נעבור ל-native.
- החלטת ה-native (Capacitor מול Flutter) נדחית עד שהליבה (RTL/אופליין/TTS/נתונים) עובדת.
