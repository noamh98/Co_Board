# ADR-0005 — רישוי ניקוד (Dicta Nakdan) + fallback ⚠️ הצעה (Phase 5)

- **סטטוס:** מוצע (לא מיושם). סיכון משפטי/ToS — לפתור לפני ייצור מסחרי.
- **הקשר:** `services/nikud/nakdanClient.ts` פונה ל-endpoint ייצור של Dicta
  (`nakdan-2-0.loadbalancer.dicta.org.il`) ללא רישיון מפורש כברירת-מחדל. Dicta הוא שירות
  אקדמי — שימוש מסחרי ללא רישיון = סיכון ToS.

## אפשרויות
1. **גידור מאחורי env + בדיקת non-prod** (מיידי, Phase 1/3): לקרוא ל-Nakdan רק כש-
   `VITE_NAKDAN_ENDPOINT` מוגדר; אחרת לדלג בחן (התווית מוקראת ללא ניקוד). מונע קריאה
   לא-מורשית בייצור עד שהרישוי מוסדר. **זהו הצעד ה-Ponytail המיידי.**
2. **רישיון מסחרי מול Dicta** (Phase 5): להסדיר הסכם; אז להחזיר את ה-endpoint כברירת-מחדל.
3. **ניקוד מקומי/precompute**: להרחיב מילון ניקוד מקומי (offline, `nikudCache`) למילים נפוצות —
   מצמצם תלות חיצונית גם ככה.

## המלצה
לאמץ (1) מיד (גידור env), ובמקביל להרחיב את מילון הניקוד המקומי (3). (2) רק אם המוצר
מצדיק. **קבלה:** ללא env → אפס קריאות ל-Dicta; ניקוד מקומי עובד offline.
