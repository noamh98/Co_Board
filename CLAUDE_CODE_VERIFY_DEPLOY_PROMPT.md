## קונטקסט
שלושת התיקונים (חיזוי מילה, סריקת שורות-עמודות, לופ speak()) אומתו ידנית בקוד המקור ותואמים לתיאור שניתן. לפני שסוגרים את המשימה צריך: לכתוב טסט ל-edge case אחד שלא טופל, לבדוק build, ולהכין ל-deploy.

## משימה 1 — Edge case בסריקת שורות-עמודות
ב-`app/src/domain/scanning/scanEngine.ts`, פונקציית `advance` בפאזת `'cell'` מתקדמת לפי `c.cols` המלא, גם כששורה אחרונה ב-grid לא מלאה (למשל 5×5=25 תאים אבל רק 23 גלויים → השורה האחרונה חסרה 2 תאים). זה גורם לסריקה לכלול אינדקסים שלא קיימים בפועל (לא קריסה — `onSelect` ב-`App.tsx` כבר עושה guard עם `if (c)` — אבל המטפל יכול "לבחור" תא ריק בלי משוב). תקנו כך שב-row-column, מספר העמודות הנסקרות בשורה האחרונה יוגבל למספר התאים שבאמת קיימים בה (יש להעביר את `itemCount` בפועל ל-`scanEngine`, לא רק `rows/cols`). הוסיפו טסט ל-`scanEngine.test.ts` לגריד לא מרובע (למשל 5 עמודות, 23 תאים).

## משימה 2 — הרצת הבדיקות וה-build
הריצו `npm test` ו-`npm run build` בתיקיית `app/`. אם יש שגיאות TypeScript או טסטים שנכשלים — תקנו לפני שממשיכים.

## משימה 5 — בדיקת רגרסיה ממוקדת בדפדפן (chrome-devtools / Claude in Chrome MCP)
פתחו את ה-URL ובדקו רק את שלושת התרחישים הבאים (לא צריך לחזור על כל ה-QA המלא):
1. **I2 — חיזוי מילה**: הפעילו את הטוגל, הקלידו/בחרו תא, ובדקו שמופיעים chips של הצעות מילה (לפחות fallback מהתאים הגלויים) — צלמו לפני/אחרי.
2. **I3 — סריקת שורות-עמודות**: בהגדרות נגישות, החליפו את "מצב סריקה" ל"שורות-עמודות", בדקו שיש הדגשת שורה שלמה, ושלחיצת מתג/Enter צוללת לתאים בתוך השורה ובוחרת תא בודד.
3. **כפתור "דבר" — בדיקת לופ**: הריצו אינטרצפט על `speechSynthesis.speak` (כמו בסשן ה-QA הקודם) ולחצו על "דבר" כמה פעמים ברצף מהיר — ודאו שמספר הקריאות תואם למספר הלחיצות ולא מתפוצץ ללופ.

תעדו PASS/FAIL לכל אחד משלושת התרחישים, ועדכנו את `HANDOFF.md` בסעיף "אומת בדפדפן בתאריך [תאריך]".
